# Phase 1 设计文档 — 最小可玩战斗

## 目标

实现双资源卡牌战斗核心 + 10 张基础卡 + 1 种敌人 + 完整的 标题→战斗→结算 流程。

## 技术栈

- Vite + TypeScript + ES Modules
- 纯 DOM 渲染，无框架
- 像素复古风 UI，桌面端优先
- GitHub Pages 部署

## 架构

```
src/
├── game/              # 纯游戏逻辑，不依赖 DOM
│   ├── types.ts       # 所有类型定义
│   ├── cards.ts       # 10 张基础卡数据
│   ├── enemies.ts     # 敌人数据和 AI 行为
│   ├── combat.ts      # 战斗状态机
│   └── effects.ts     # 卡牌效果执行
├── ui/                # UI 渲染层
│   ├── renderer.ts    # 主渲染器
│   ├── scenes/
│   │   ├── title.ts   # 标题画面
│   │   ├── battle.ts  # 战斗画面
│   │   └── result.ts  # 结算画面
│   └── animations.ts  # CSS 动画
├── main.ts            # 入口
└── style.css          # 全局样式 + 像素字体
index.html
```

核心原则：`game/` 纯逻辑不依赖 DOM，`ui/` 负责渲染和用户交互转发。

## 核心类型

- `BattleState`：战斗完整快照（玩家状态、敌人状态、回合数、阶段）
- `Card`：卡牌（名称、费用、资源类型、类别、稀有度、效果列表）
- `CardEffect`：联合类型（damage / armor / heal / gain_mana / gain_stamina / burn）
- `EnemyIntent`：敌人意图（attack / defend / buff）

## 敌人：地精散兵

- HP: 20
- 行为循环：攻击 6 → 攻击 6 → 战吼（自身+2力量）→ 重复
- 力量效果：每层力量使攻击伤害+2

## 战斗流程

1. 回合开始：刷新资源（体力→3，魔力→2）、护甲清零、灼烧结算、抽5张牌、显示敌人意图
2. 玩家回合：点击手牌打出（检查资源）、效果立即结算、检查胜负
3. 敌人回合：执行意图、检查胜负、推进行为索引、手牌进弃牌堆
4. 抽牌堆空时弃牌堆洗牌放回

## 三个场景

1. **标题画面**：游戏名称 + "开始冒险" 按钮
2. **战斗画面**：上方敌人区、中间玩家状态栏、下方手牌区 + 结束回合按钮
3. **结算画面**：胜利/失败 + 战斗统计 + "再来一局" 按钮

## UI 风格

- 像素复古风，CSS 像素字体 + emoji 图标
- 桌面端优先
- 简单 CSS 动画：伤害数字飘出、卡牌打出淡出、HP 条过渡
