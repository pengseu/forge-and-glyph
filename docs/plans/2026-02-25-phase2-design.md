# Phase 2 设计文档 — Roguelike 感（路线图 + 战后选卡）

## 目标

实现Roguelike地牢探索的核心体验：路线图导航 + 战后选卡构筑 + 多敌人 + Boss战。

## 核心设计决策

### 架构方案：最小改动

复用Phase 1的战斗系统，在上层添加路线图和选卡UI。状态流转：

```
标题 → 路线图 → 战斗 → 选卡 → 路线图 → ... → Boss战斗 → 结算
```

### 数据结构

- `RunState`：当前冒险的全局状态（路线图、位置、卡组、敌人进度）
- `MapNode`：路线图节点（类型、敌人、是否已完成）
- `RewardState`：选卡界面的状态（3张候选卡、玩家选择）

## 路线图系统

### 节点类型（第一幕）

| 节点类型 | 数量 | 敌人 | 奖励 |
|------|------|------|------|
| 普通战斗 | 5 | 随机普通敌人 | 1张随机卡 |
| 精英战斗 | 2 | 强力敌人 | 2张随机卡 |
| Boss | 1 | 地精王 | 1张特殊卡（固定） |

**总计8个节点**，分支结构，玩家可选路径。

### 路线图生成

固定分支结构示例：
```
[普通] → [普通] → [精英] ↘
                          [Boss]
[普通] → [普通] → [普通] ↗
```

保证每条路径都能到达Boss。

### UI

- 显示节点网络（网格或树形）
- 标记当前位置
- 标记已完成节点
- 点击未完成节点进入战斗

---

## 战后选卡系统

### 流程

1. 战斗胜利 → 显示3张随机卡牌
2. 玩家选1张或跳过
3. 卡牌加入卡组（如果选择）
4. 返回路线图

### 卡牌池

- **普通战斗**：从20张卡中随机3张
- **精英战斗**：从20张卡中随机3张（可能包含稀有卡）
- **Boss战斗**：固定掉落1张特殊卡（不可跳过）

## 敌人库

### 普通敌人（3种）

#### 地精散兵（已有）
- HP: 20
- 行为：攻击6 → 攻击6 → 战吼(+2力量) → 重复

#### 森林狼（新增）
- HP: 15
- 行为：首回合嚎叫(全体友方+2力量) → 之后每回合攻击8

#### 蘑菇怪（新增）
- HP: 25
- 行为：释放孢子(2中毒) → 攻击5 → 重复

### Boss

#### 地精王
- HP: 100
- 被动：每个地精+3攻击力
- 阶段1（>50% HP）：召唤 → 重击12 → 战吼
- 阶段2（≤50% HP）：不召唤，每回合双击，攻击力 = 12 + 地精数×3

## UI场景

### 新增场景

- **map**：路线图显示
  - 显示节点网络
  - 显示当前位置
  - 点击节点进入战斗

- **reward**：战后选卡界面
  - 显示3张候选卡
  - 选择按钮 + 跳过按钮

### 修改场景

- **battle**：添加"返回路线图"的逻辑（战斗结束后）
- **result**：改为"冒险结束"（通关或全灭）
