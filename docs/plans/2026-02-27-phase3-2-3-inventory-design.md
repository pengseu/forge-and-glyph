# Phase 3.2 + 3.3 + 背包设计文档

## 目标

在现有主线基础上完成：
- 3-2 商店与金币系统稳定化
- 3-3 材料与锻造首版
- 地图顶部常驻背包入口，支持装备切换

## 设计范围

- 保持当前纯函数 game 层 + DOM 场景层架构。
- 所有库存类数据统一落在 `RunState`，避免分散状态。
- 背包作为独立场景 `inventory`，从 map 顶部按钮进入。
- 背包只读展示卡牌/材料/武器，武器支持“设为当前装备”。

## 数据结构

`RunState` 增加：
- `gold: number`
- `materials: Record<MaterialId, number>`

新增：
- `MaterialId` 类型与材料定义表（图标、名称、上限）
- `ShopOffer`（已存在）继续作为商店 UI 数据

## 玩法流转

1. 战斗胜利：
- 发放金币
- 进入奖励页
- 非 Boss：二选一（卡牌三选一 / 材料奖励）
- Boss：卡牌奖励 + 额外 Boss 材料

2. 商店：
- 买卡（3 张报价）
- 回复 30% HP（30 金币）
- 移除 1 张卡（50 金币）

3. 锻造：
- 新增铁匠节点与场景 `forge`
- 消耗材料打造武器（首版先做 2-3 把）
- 打造后进入背包可切换装备

4. 背包：
- 从 map 顶栏进入
- 展示卡牌清单（按 defId 聚合计数）
- 展示材料与上限
- 展示武器并支持“装备”

## 测试策略

- Run 纯逻辑：金币、材料上限、锻造消耗、装备切换
- Map：新增铁匠节点存在与可达性
- Reward：材料奖励流程最小验证
- 全量回归：`npm test` + `npm run build`

## 非目标（本轮不做）

- 材料战斗内应急使用
- 多武器复杂战斗特性
- 附魔与共鸣（下一阶段）
