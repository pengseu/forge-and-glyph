# Phase 3 Step 6 设计（40卡池 + 第一幕7层 + 未知事件）

## 目标
在现有 Phase 3（精英/商店/材料/锻造/附魔）基础上，完成完整第一幕可玩闭环：
- 卡池扩展到 40 张并接入升级
- 地图改为 7 层分叉结构并覆盖关键节点
- 引入最小可用未知事件系统（5类事件）
- 保持现有战斗、奖励、商店、附魔体系兼容

## 范围与分阶段
按 `TODO_PHASE3.md` 执行任务 A→G：
- A：卡牌数据层（13张定义 + 升级映射 + 数据测试）
- B：卡牌效果实现层（新增效果 + 状态联动 + 战斗测试）
- C：奖励与商店接入（卡池来源 + 稀有度/价格回归）
- D：7层地图重构（拓扑 + 可达性 + 约束）
- E：未知事件系统（数据结构 + 场景 + 状态写回）
- F：整局联调（从开局到 Boss）
- G：Phase3 总验证（核心循环/构筑/经济/路线/体验）

## 架构策略
- 数据优先：先让 `cards.ts` + `campfire.ts` 完整描述新卡，再推进行为实现。
- 规则模块化：新增效果尽量复用 `effects.ts` 既有分派，避免在 `combat.ts` 硬编码。
- 地图与事件分层：`map.ts` 负责节点与连线，事件池与结果结算放在 `run.ts`/新模块。
- UI 最小接入：先保证流程可达与反馈清晰，复杂动效后置。

## 关键约束
- 严格 TDD：每个新增行为先有失败测试，再补最小实现。
- 任务顺序固定：A 完成后才进入 B，依次推进。
- 不回退已完成 Step2-5 能力，保证全量回归通过。

## 测试策略
- 数据层：`cards.test.ts` + `campfire.test.ts` 增加存在性与字段完整性校验。
- 行为层：`effects.test.ts` / `combat.test.ts` 补关键组合与边界。
- 系统层：`map.test.ts`、`run.test.ts`、场景单测覆盖流程连通。
- 验收层：全量 `npm test` + 手动 run 冒烟。
