# 战斗敌人布局优化设计

日期：2026-03-06  
范围：仅战斗场景敌人 UI 布局与信息层级，不改战斗逻辑与数值。

## 背景

当前敌人信息集中堆在立绘顶部：意图、血条、状态、护甲、详细说明同时向上叠加，导致视觉重心过高，立绘本身反而像配角。

## 目标

- 保留现有战斗舞台构图与敌人绝对定位
- 优先解决“头重脚轻”与重复说明
- 不依赖新增美术素材，首版用纯 CSS 落地
- 兼容 1~4 个敌人场景与现有目标选择交互

## 方案

采用“顶部轻量 + 底部锚定 + 说明浮层”的结构：

1. 顶部仅保留 `enemy-intent`
2. 血条与护甲合并为一条 `enemy-vitals`
3. 名字改为立绘底边纸签，不再占据垂直流式布局
4. 状态徽章移到立绘下沿，作为底部稳定配重
5. 被动与详细意图进入 `enemy-tooltip`，通过 hover/focus 展示
6. 移除常驻 `enemy-intent-hint` 文本，避免与 tooltip 重复

## 落地边界

- `src/ui/scenes/battle.ts`：抽出敌人块 HTML 生成函数，便于单测
- `src/ui/styles/batch2-core.css`：调整敌人布局、定位和可见性
- `src/style.css`：仅保留旧样式兜底，不新增主要规则

## 验收

- 敌人顶部常驻信息减少为“意图 + 细血条/护甲”
- 名字与状态不再堆在头顶
- 详细说明默认隐藏，但键鼠 hover/focus 可读
- 不影响敌人点击、目标预览、死亡态与已有图片加载规则
