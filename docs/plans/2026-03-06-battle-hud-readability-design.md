# 战斗 HUD 可读性优化设计（中改版）

日期：2026-03-06  
范围：仅战斗场景顶栏信息结构与可读性，不改战斗数值逻辑。

## 1. 背景与问题

当前战斗界面在视觉上存在“廉价感”，核心原因之一是顶栏信息没有优先级分层：

- 关键读数（HP/资源/回合）与次要信息（试炼、附魔反馈、状态说明）混排
- 字号和对比层级不清，主次信息同时抢注意力
- 状态徽章堆叠在主视区，造成类似“调试输出”的观感

目标是在不重做整套美术主题的前提下，提升读数效率与成品感。

## 2. 目标与非目标

目标：

- 0.3 秒内可读到“生存状态 + 可行动资源 + 当前回合”
- 减少顶栏信息噪音，避免一行塞满
- 保持现有纸质手作风格变量体系（tokens）不被破坏

非目标：

- 不重做战斗主舞台构图（角色/敌人尺寸与位置后续单独优化）
- 不改卡牌系统视觉规范
- 不引入新资源依赖或新字体文件

## 3. 信息架构方案（已确认）

采用 `1 主栏 + 1 副栏` 的两层 HUD：

1. 主栏（始终可读，优先级最高）
`HP 条 + 护甲 + 体力 + 法力 + 武器 + 回合 + 操作按钮`

2. 副栏（战术信息，低强调）
`试炼文本 + 附魔反馈 + 状态徽章汇总 + 状态说明入口`

3. 状态策略
- 主栏仅保留“危险状态提示”（如中毒/虚弱）
- 其余状态统一进入副栏小徽章

## 4. 视觉可读性规范（已确认）

### 4.1 字号层级

- 核心数值（HP/体力/法力/护甲）：14px，`var(--font-number)`
- 主栏标签（武器/回合）：10-11px，`var(--font-body)`
- 副栏说明（试炼/附魔反馈）：9-10px，`var(--font-body)`

### 4.2 对比层级

- 主栏背景：`var(--paper-cream)`，文字：`var(--ink-primary)`
- 次要说明统一 `var(--ink-secondary)`，避免过浅灰
- 危险信息才使用 `--color-hp / --color-danger`

### 4.3 空间节奏

- 主栏组间距：12px
- 组内间距：6px
- 轻分隔：1px 虚线/浅色竖线
- 顶栏高度：52px -> 60px

### 4.4 数字稳定性

- 资源数字统一用 `var(--font-number)`（等宽感）
- 关键数字容器设置最小宽度，避免位数变化引发布局抖动

## 5. 落地结构（已确认）

## 5.1 DOM 调整（battle.ts）

- 保留 `.player-bar` 容器
- 增加 `.hud-main` 与 `.hud-sub`
- 主栏放入：`hp/maxHp`、`armor`、`stamina/maxStamina`、`mana/maxMana`、`weaponText`、`turn`
- 副栏放入：`trialText`、`enchantText`、`enchantFeedback`、非危险状态徽章

### 5.2 CSS 调整范围

- 主改：`src/ui/styles/batch2-core.css`（战斗 HUD 布局与层级）
- 补充：`src/ui/styles/batch5-polish.css`（低血量/危险状态高亮与过渡）
- 不改全局 token 与基础主题文件，避免牵连 map/reward/title 场景

### 5.3 实施顺序

1. 完成 DOM 分层，不改视觉语义
2. 套用字号/对比/间距规范
3. 添加危险状态高亮规则
4. 回归检查 battle/map/reward 的 `.player-bar` 复用影响

## 6. 验收标准

- 战斗界面主栏信息不再横向溢出或拥挤断行
- 玩家可在短时间内识别 HP、资源、回合与武器
- 副栏存在但不喧宾夺主，状态密集时仍不破坏主栏可读性
- 不改变现有战斗逻辑与交互结果
