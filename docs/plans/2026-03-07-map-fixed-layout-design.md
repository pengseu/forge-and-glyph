# 地图页固定纸板布局设计

**日期：** 2026-03-07  
**目标：** 将地图页改成左右两张独立纸板的固定布局，解决当前地图纸、内容区域与右侧说明层互相牵连的问题。

## 背景
当前地图页虽然已经解决了主要贴图拉伸问题，但布局层级仍然偏“流体”：
- 左侧地图纸、真实地图内容和滚动区域仍然耦合在一起
- 地图底图会跟随内容区域大小变化，导致视觉锚点不稳定
- 右侧纸板和文字虽然已经部分解耦，但仍缺少明确的“纸图层”和“内容层”边界

用户希望将地图页收敛为更稳定的舞台式布局：
- 左侧一张固定宽度的地图底纸
- 只有纸面中央的小窗口用于展示与滑动地图
- 右侧一张固定宽度的信息纸板
- 文字始终独立定位，不受背景纸张比例影响

## 设计目标
1. 左侧区域采用固定视觉容器：背景图宽 800px，高度自适应。
2. 地图真实可见区只在居中的 582×345px 区域内显示，并支持滑动。
3. 右侧区域采用固定视觉容器：背景图宽 300px，高度自适应。
4. 右侧背景图与文字层彻底解耦，文字仅通过独立内容容器定位。
5. 保留现有地图节点逻辑、路径逻辑、hover/点击逻辑，不重写玩法数据结构。

## 方案对比

### 方案 A：固定纸板 + 内部视窗滑动区（推荐）
- 左侧 `map-stage` 变成固定宽 800px 的纸板展示区。
- 在纸板内部放置一个绝对定位的 `map-viewport`，尺寸固定为 582×345px。
- 原有 `map-scroll` 和 `map-canvas` 移入 `map-viewport` 内部。
- 右侧 `map-side-panel` 变成 300px 宽的纸板展示区，背景图仅负责装饰。
- 右侧文字层使用独立的 `map-side-content` 绝对定位。

**优点：**
- 最贴近用户预期
- 背景与内容职责清晰
- 节点和路径不需要额外缩放
- 后续替换素材时，容器规则稳定

**缺点：**
- 需要重新组织地图场景 DOM 结构
- 需要重新计算自动聚焦/初始滚动位置

### 方案 B：保留现有滚动区，只加遮罩裁切
- 继续保留当前左侧整体滚动区
- 在外层叠一张 800px 纸板图，并用遮罩限制中间显示窗为 582×345px

**优点：**
- 改动少

**缺点：**
- 结构上仍然是耦合的
- 后续调素材和调层级时容易再次出问题

### 方案 C：把地图节点重新排版进 582×345 坐标系
- 直接将所有节点、路径、纸面都绑定到固定尺寸舞台里

**优点：**
- 视觉最统一

**缺点：**
- 改动大
- 会影响地图生成与后续扩展
- 不符合“只改展示层”的最小改动原则

## 最终方案
采用 **方案 A**。

## 结构设计

### 左侧地图区
- 外层：`map-stage-shell`
  - 固定宽 `800px`
  - 高度按背景图比例自适应
  - 只负责展示地图纸板贴图
- 内层：`map-viewport`
  - 绝对定位在纸板中央
  - 固定尺寸 `582px × 345px`
  - `overflow: hidden`
- 视窗内部：`map-scroll`
  - 负责拖拽和滚动
  - 只在视窗区域内可见
- 内容层：`map-canvas`
  - 继续承载路径、节点、互动逻辑

### 右侧说明区
- 外层：`map-side-panel`
  - 固定宽 `300px`
  - 背景图单独作为 `map-side-art`
- 内容层：`map-side-content`
  - 绝对定位
  - 不依赖背景图尺寸
  - 当前使用左右 40px、上 100px 的文字安全区

## 数据与交互
- `renderMap` 继续生成节点、路径、右侧说明内容。
- 自动聚焦逻辑要从“滚动容器可见区域”改为“582×345 视窗中心”。
- 点击节点、hover 节点、键盘交互保持不变。

## 测试策略
1. 纯函数测试：
   - 固定视窗尺寸
   - 固定纸板尺寸
   - 右侧文字区样式值
2. 场景结构测试：
   - 左侧存在固定 `800px` 外壳
   - 左侧存在 `582×345` 地图视窗
   - 右侧存在独立纸图层和独立文字层
3. 构建验证：
   - `npm test -- src/ui/scenes/__tests__/map-unit.test.ts`
   - `npm run build`

## 风险与约束
- 如果左侧 `map-board-forest` 新素材的实际比例与当前视觉预估不同，需要继续微调内部视窗的绝对定位值。
- 如果右侧纸板的可写区域和当前 40px / 100px 安全区不一致，应优先改内容层定位，而不是去拉伸纸图。
- 所有图片继续遵守仓库规则：容器定宽高，图片使用 `object-fit: contain` 或 `cover`。
