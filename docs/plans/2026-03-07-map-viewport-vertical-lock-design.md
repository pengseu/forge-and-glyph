# 地图页视窗纵向锁定与节点轻量化设计

**日期：** 2026-03-07  
**目标：** 在固定纸板地图布局上继续收敛交互：左侧视窗只允许上下滑动，半透明纸感层固定不跟内容滚动，节点去掉白底卡片样式。

## 用户确认
采用方案 A：
- 保留多分支地图结构
- 地图内容整体在视窗中水平居中
- 不再需要左右滑动
- 节点改成无白底的轻量视觉

## 设计要点
1. **半透明层固定**
   - 当前视窗里的半透明效果如果挂在滚动内容上，会随着地图上移一起移动。
   - 改为挂在 `map-viewport` 或其固定 overlay 上，只作为静态遮罩层。

2. **只保留纵向滚动**
   - 节点的横向分布继续保留，但要整体压缩并居中到 `582px` 视窗宽度内。
   - `map-scroll` 改为 `overflow-y: auto`，隐藏横向滚动。
   - 初始聚焦逻辑改为只算 `scrollTop`，`scrollLeft` 固定为 0。

3. **节点去白底**
   - 取消节点外层白色卡片背景。
   - 保留节点徽章图、状态环和文字标签。
   - Hover/当前节点高亮仍保留，但只用阴影/描边/环，不再用白色块面。

## 测试策略
- 纯函数测试：
  - 地图内容水平拟合到视窗宽度的 helper
  - 纵向聚焦 helper
- 样式/结构测试：
  - 视窗 overlay 为固定层
  - 节点样式不再依赖白底面板
- 构建验证：
  - `npm test -- src/ui/scenes/__tests__/map-unit.test.ts`
  - `npm run build`
