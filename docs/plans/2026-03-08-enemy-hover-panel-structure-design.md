# 敌人 Hover 信息面板结构设计（2026-03-08）

## 目标

优化战斗页敌人信息的阅读层级：默认态只保留战斗决策最需要的读数，hover 态在怪物立绘右侧展开完整敌情纸签，承载解释性信息。

## 设计原则

1. 默认态服务“快速决策”，hover 态服务“完整理解”
2. 不再把解释性文字堆在敌人头顶
3. 右侧面板固定为统一结构，避免不同敌人信息块忽长忽短
4. 信息优先级固定：怪名 > 血量护甲 > 意图 > 状态 > 被动

## 默认态保留内容

- 头顶：意图读数
- 中部：血条与护甲简写
- 底部：状态徽标行
- 名字保留为轻量纸签

## Hover 态面板结构

### 第一块：怪名
- 单独一行
- 使用较高字重的纸签标题

### 第二块：血量与护甲
- 放在同一行
- `血量：当前/上限`
- `护甲：数值`（无护甲则隐藏）

### 第三块：意图说明
- 第一行：意图标题，例如“攻击 / 护甲 / 召唤 / 凝视 / 重击”
- 第二行：意图详细描述，直接复用当前 `intentHint`

### 第四块：状态列表
- 竖排，一条状态一行
- 每条显示：状态名称 + 层数
- 无状态时显示“当前无额外状态”

### 第五块：被动描述
- 仅在该敌人存在被动时显示
- 位于面板最底部，作为解释性文字区块

## 样式建议

- 面板固定在敌人立绘右侧
- 宽度控制在 `180~220px`
- 使用牛皮纸/纸签视觉，不做现代浮层发光
- 分区之间使用虚线或针脚线分隔
- hover/focus 时淡入；默认完全隐藏

## 本轮实现边界

- 先完成结构与层级，不强依赖新增素材
- 状态区先以“中文名 + 数值”为主，后续再接入图标素材也不破坏结构
- 不在本轮重做敌人头顶意图徽标资源，只先处理 hover 面板内容组织
