# 敌人三怪 / 四怪布局方案

## 目标
在不破坏当前剪纸森林战斗页层次的前提下，让 3 敌 / 4 敌场景保持：
- 立绘可辨认
- 顶部意图不互相打架
- hover 右侧信息卡仍可用
- 前景草丛和手牌盒不被频繁遮挡

## 方案 A：三怪布局（推荐默认）

### 适用条件
- 场上存活敌人 = 3
- 仍希望保留较完整立绘比例

### 布局参数
- 敌人整体区域横向分为 3 列
- 单列容器宽度：战斗区可用宽度的 `28% ~ 30%`
- 相邻敌人中心点间距：`210px ~ 235px`
- 敌人立绘缩放：以当前双敌基准的 `0.88 ~ 0.92`
- 中间敌人略后退：`translateY(8px)`
- 两侧敌人略外摆：左 `-8deg`，右 `8deg` 的极轻微视觉倾向即可，不要真旋转容器

### 信息排布
- 头顶：只放 `意图 icon + 数值/短词`
- 名字：仍贴底放在怪物本体下沿
- 状态：底部纸条允许单行换行，最多两行
- hover：统一弹右侧高纸卡，不在头顶堆信息

### 优点
- 三怪仍能看清立绘轮廓
- 中间敌人天然形成视觉主目标
- 和双敌布局衔接最自然

## 方案 B：四怪布局（推荐切换方案）

### 适用条件
- 场上存活敌人 >= 4
- 横向拥挤开始明显影响意图与状态阅读

### 布局参数
- 横向分为 4 列，但不要均匀铺满，整体向左压缩一点，给 hover 信息留右侧呼吸区
- 单列容器宽度：战斗区可用宽度的 `21% ~ 23%`
- 相邻敌人中心点间距：`152px ~ 176px`
- 敌人立绘缩放：以当前双敌基准的 `0.72 ~ 0.8`
- 最外侧两个敌人上移 `4px`，中间两个下移 `4px`，做出轻微错层

### 信息排布
- 头顶：只放 `意图 icon + 一个核心数字/短词`
- 血条：保持窄条，不显示数值
- 名字：压缩成更窄的纸条，字号下降 1 级
- 状态：底部纸条强制两行流式排列；超过 4 个状态时，hover 面板里完整展示，默认态不全量展开
- hover：仍只允许一个敌人提升到最高层，右侧面板固定在怪物右侧或战场右侧，不跟着横向挤压

### 优点
- 比“继续放大 UI”稳定得多
- 默认态保持可扫读，详细信息交给 hover
- 不会因为 4 个敌人而把顶部做成一排小字地狱

## 切换建议
- `1~2 敌`：当前已实现布局
- `3 敌`：切到方案 A
- `4 敌及以上`：切到方案 B

## 实现建议
- 不要用纯 `scale()` 粗暴压缩整组敌人，优先分别控制：
  - 敌人槽位宽度
  - 敌人 sprite 容器宽高
  - 列间距
  - 名字 / 状态条字号
- hover 信息卡保持统一尺寸，不要跟敌人数一起缩小
- 敌人状态条优先允许换行，不要继续横向硬塞
