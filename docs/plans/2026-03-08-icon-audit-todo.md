# UI icon / emoji 审计清单（2026-03-08，已更新）

> 范围：`src/ui/scenes/*`、`src/ui/renderer.ts`、`src/main.ts`
> 说明：本清单只统计“玩家可见的文本 icon / emoji / 符号化占位”；材料图、状态徽标图、敌人意图素材图这类功能素材不在清理范围内。

## 当前结论

- 玩家可见的文本 `emoji` 已清空
- 已素材化系统不受影响：敌人意图、敌人状态、玩家状态、材料图、武器图、地图节点底图等均保留
- 当前剩余问题已从“emoji 清理”转为“少量文字型占位是否继续素材化”

## 已完成项

- 标题页、武器选择页、地图页、战斗页、商店页、奖励页、背包页、篝火页、附魔页、幕间页中的文本 `emoji` 已完成清理
- 地图节点单字覆盖层已移除，改为只保留节点素材底图 + 文字标签
- 地图顶部状态栏已收口为“标签 + 数值”，不再使用符号化图标占位
- 奖励页武器掉落语义已固定为“自动入背包，可立即装备”

## 仍值得继续关注的文字占位

### P1

- `src/ui/scenes/enchant.ts`
  - 附魔槽位当前仍使用单字：`烈 / 冰 / 雷 / 魂 / 虚 / 祝 / 渊`
  - 这不是 emoji，但属于明显的文字型图标占位
  - 建议：后续若补齐附魔徽记素材，再替换为小型徽记图

- `src/ui/scenes/map.ts`
  - 顶部武器栏在缺少武器缩略图时，仍会回退到短字：`杖 / 匕 / 锤 / 弓 / 刃`
  - 建议：后续继续补武器缩略图，减少 fallback 出现概率

- `src/ui/card-cost.ts`、`src/ui/scenes/battle.ts`、`src/ui/scenes/shop.ts`
  - 卡牌费用角标当前使用文字型功能标签
  - 建议：若后续统一卡牌视觉，可以单独做一套费用纸签素材

### P2

- `src/ui/scenes/style-lab.ts`
  - 若仍保留 mock 文案页，可能存在少量展示性占位文字
  - 建议：逐步改成真实页面片段，减少 mock 与正式页分叉

## 建议保留为纯文字的部分

- 页面标题
- 普通按钮与页签
- 金币、牌组、幕数这类短标签
- `关闭`、`已升级`、`立即装备` 这类动作或状态文案
