# 奖励页选择规则设计（2026-03-08）

## 最终规则

奖励页采用三层语义：

### 1. 自动获得区
这部分在进入奖励页前已经结算完成，只负责展示，不参与选择：
- 金币
- 战后恢复
- Boss 材料（如王冠碎片 / 暗影水晶 / 深渊之心）
- 其他明确标记为“已自动获得”的内容

### 2. 主奖励单选区
这部分是玩家本次战斗后的主决策区，只能二选一：
- 某一张卡牌
- 本场全部可选材料组成的“材料包”

即：
- 不是“从所有卡牌和材料单件里选五选一”
- 也不是“固定拿全部材料再选一张卡”
- 而是“卡牌成长”与“资源成长”二选一

### 3. 额外掉落区
这部分与主奖励单选区解耦：
- 武器掉落
- 当前可作为额外处理项展示在下方，不混入主奖励选择

## 选择这套规则的原因

### 为什么不用“所有内容只能选一个”
- 材料可能在一场战斗里出现多种
- 若拆成单件材料牌，会把奖励区切得很碎
- 玩家会被迫比较大量零散价值，不利于形成清晰的成长选择

### 为什么不用“固定拿全部材料再选卡”
- 资源和构筑会同时增长，奖励膨胀太快
- 会削弱每次战斗奖励的取舍感

### 为什么采用“材料包 vs 单卡”
- 卡牌代表构筑方向
- 材料代表资源方向
- 奖励页的决策会变成明确的战略选择，而不是杂项拣货

## UI 约束

- 上层：卡牌选项 + 1 个材料包选项，统一置于同一条选秀带中
- 下层：自动获得区 / 武器掉落区
- 自动获得内容不能伪装成可选奖励
- 武器掉落不应与主奖励竞争视觉优先级

## 后续建议（未立即实现）

- 这样语义会比“掉落了但不点就消失”更统一
