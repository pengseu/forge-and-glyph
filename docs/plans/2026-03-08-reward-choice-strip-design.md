# 奖励页单行选秀设计

## 目标

将奖励页改为两层结构：
- 上层只放“需要玩家选择”的奖励，并统一为单行选秀区
- 下层只放“无需玩家选择”的自动结算内容

## 设计结论

### 1. 主奖励区
- 卡牌奖励与材料奖励同层展示
- 统一使用横向比较逻辑，降低决策成本
- 玩家只需要回答“这次拿什么”，而不是先理解奖励分区

### 2. 自动结算区
- 自动获得
- 武器掉落 / 装备
- 以上内容不参与主奖励比较，只做补充说明与后续操作

### 3. 交互原则
- 卡牌与材料都属于可选奖励，选中后应共享一套高亮 / 置灰逻辑
- 保持当前回调语义不变：卡牌仍走 `onSelectCard`，材料仍走 `onSelectMaterialReward`
- 自动获得与装备武器维持现有非互斥逻辑

## 最小实现策略
- 不重构奖励系统数据结构
- 只在渲染层新增“材料选择卡”与“统一选秀条” helper
- 复用现有奖励页动画、纸片风和下方补充区样式
