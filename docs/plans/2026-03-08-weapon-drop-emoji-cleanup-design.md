# 武器掉落语义与文本 Emoji 清理设计（2026-03-08）

## 目标

一次性收口两类体验问题：
- 奖励页中的武器掉落语义不清，改为“自动进背包，可立即装备”
- 全局页面中的文本 emoji 统一清理，但不影响已素材化的图示系统

## 规则一：武器掉落

### 最终规则
- 战斗胜利后若存在 `droppedWeaponId`，武器立即进入背包
- 奖励页下方额外区只负责展示该武器，并提供“立即装备”按钮
- 玩家不点击“立即装备”也不会丢失武器

### UI 语义
- 主奖励区：仍是“卡牌 or 材料包”二选一
- 自动获得区：展示必得内容
- 额外掉落区：展示“已收入背包”的武器，并允许立即装备

### 这样做的原因
- 避免当前“看起来掉落了，但不点就消失”的歧义
- 武器掉落从临时交互项变成稳定资产，更符合玩家直觉
- 不让武器掉落与主奖励选择发生语义混淆

## 规则二：文本 Emoji 清理

### 清理范围
只清理文本里直接写死的 emoji / icon，不动任何已换成图片素材的内容。

### 清理对象
- 页面标题
- tab 文案
- 按钮前缀
- 价格 / 金币 / 花费文案
- 服务标题 / 子标题
- 说明性标题文本
- 地图节点与摘要中仍用 emoji 的文本标记
- 战斗内仍属纯文本说明的 emoji（如状态说明弹窗、普攻按钮等）

### 不清理对象
- 材料图片素材
- 状态徽标素材
- 敌人意图素材
- 其他已经走图片资源的视觉元素

## 建议文案策略

- 金币价格：统一改为 `25 金币`
- 金币余额：统一改为 `余额：188`
- 页面标题：保留纯文字，如 `旅途商店`、`背包`、`篝火休憩`
- 服务标题：改为 `治疗`、`删卡`、`变卡`
- 地图节点：若原来依赖 emoji 区分，改为短中文标签或已有素材方案

## 本轮实现范围

### 必做
- 奖励页武器自动入包
- 奖励页武器区文案改为已入包语义
- 清理商店 / 奖励 / 背包 / 篝火 / 附魔 / 标题 / 武器选择 / 幕间 / 地图 / 战斗说明里的文本 emoji

### 暂不做

- 新增一套专门金币图标素材
