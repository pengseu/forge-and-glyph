# V3.1 战斗与事件收口设计

## 目标

基于 `V3.1` 分支收口 5 个体验问题：

- 铁匠的附魔在 hover 时显示具体效果。
- 敌人较多时，战斗页敌方编队更紧凑，避免遮挡和错位。
- 地精术士的行动意图标题不再显示为“行动”。
- 神秘商人等事件获得卡牌后，在事件页给出轻提示。
- 第二幕战斗页面的元素位置先回退到第一幕的稳定布局。

## 现状

- 铁匠场景 `src/ui/scenes/forge.ts` 目前只渲染附魔按钮，没有可读的效果说明。
- 敌方区域在 `src/ui/scenes/battle.ts` 维持单一 DOM 结构，但 CSS 主要围绕第一幕绝对定位展开；人数增多时仍容易挤压。
- 敌人 hover 面板标题来自 `resolveEnemyIntentDetailTitle()`，目前未覆盖治疗和友方强化语义。
- 事件系统 `src/game/events.ts` 只返回运行态，不返回用于 UI 的奖励摘要；`src/main.ts` 结算后直接离开事件页。
- 第二幕战斗存在布局异常，但当前第一幕布局已经可用，应优先复用其层级与坐标体系，而不是继续维护失稳的第二幕定位。

## 方案

### 1. 铁匠附魔 hover 说明

- 在 `forge.ts` 为每个附魔按钮输出可读的效果文案与数据属性。
- 增加一个场景内的附魔说明面板，hover 或 focus 某个附魔按钮时同步刷新面板内容。
- 说明内容直接复用 `src/game/enchantments.ts` 中的定义文本，避免 UI 与规则文案分叉。
- 当双槽附魔可触发共鸣时，说明面板同时展示当前 hover 附魔与现有槽位可能形成的共鸣名称，帮助玩家预判替换价值。

### 2. 敌人较多时的布局优化

- `battle.ts` 根据当前存活敌人数给敌方容器输出数量类，例如 `enemy-area--count-1` 到 `enemy-area--count-4`。
- `batch2-core.css` 保留现有第一幕稳定定位，但把 3 敌与 4 敌的紧凑布局收口到数量类上，而不是靠 `nth-child` 的单一静态坐标硬撑所有情况。
- 重点保证：
  - 意图图标、血条、底部状态纸条不互相压住。
  - hover 详情面板仍能在顶部安全展开。
  - 单体与双体布局视觉不回退。

### 3. 地精术士行动意图语义修正

- 在 `resolveEnemyIntentDetailTitle()` 中补上治疗和友方增益识别。
- 治疗类显示为“治疗”或“援护”，友方强化显示为“强化”。
- 保持其他敌人的标题推导逻辑不变，避免大面积文案回归。

### 4. 事件得卡轻提示

- 扩展 `resolveEventOption()` 的返回结构，在获得卡牌时附带 `uiNotice` 或 `rewardSummary` 文案。
- 提示文案按结果生成，例如：
  - `已获得稀有卡【寒霜箭】`
  - `已获得 2 张卡牌`
  - `已获得传奇卡【王冠之刃】并加入 2 张诅咒`
- `main.ts` 在事件结算后将该提示保存到事件场景态，并在当前事件页展示一次；玩家不需要额外点击确认。
- `event.ts` 在选项区附近渲染轻提示条，风格沿用当前事件面板，不引入全局 toast 系统。

### 5. 第二幕战斗布局回退

- 第二幕的战斗画面元素布局先与第一幕共享同一套稳定规则。
- 背景、前景图资源继续保留第二幕素材；只回退“元素与单位位置体系”。
- 这样可快速消除第二幕错位，同时不给后续第三幕或深渊场景引入额外特殊分支。

## 数据流

### 事件提示

1. `resolveEventOption()` 解析事件选项。
2. 若本次选项新增卡牌，则生成面向 UI 的摘要文案。
3. `main.ts` 接收摘要，将其挂到当前事件渲染状态。
4. `renderEvent()` 渲染轻提示。
5. 玩家离开事件页后提示自然消失，不进入全局持久状态。

### 战斗布局

1. `battle.ts` 读取当前敌人数。
2. 输出对应数量类和幕类。
3. CSS 同时读取幕类与数量类，决定敌方单位排布。
4. Act 2 临时与 Act 1 共享定位体系。

## 错误处理与边界

- 若附魔定义缺少说明文本，铁匠说明面板回退为附魔名称，不阻塞交互。
- 若事件没有实际获得卡牌，则不显示轻提示。
- 若多张卡同时获得，优先输出摘要，不强行列出全部名称导致文案过长。
- 若战斗中敌人数量动态变化，数量类根据当前存活敌人重新渲染，避免击杀后仍维持拥挤布局。

## 测试策略

- `src/ui/scenes/__tests__/battle-hud-unit.test.ts`
  - 补地精术士意图标题语义。
  - 补敌人数量类或对应 HTML 断言。
- `src/ui/scenes/__tests__/event-unit.test.ts`
  - 补事件轻提示渲染。
  - 补铁匠附魔说明面板静态 HTML。
- `src/game/__tests__/events.test.ts`
  - 补神秘商人、古老图书馆、深渊裂隙等得卡事件的提示摘要。
- 运行相关 `vitest` 子集做回归。

## 不做的事

- 这次不做全局 toast 系统。
- 这次不重写整套战斗场景 DOM。
- 这次不为第二幕重新单独设计一份全新定位稿。
